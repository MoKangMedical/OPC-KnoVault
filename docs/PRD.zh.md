# OPC Trust Market / Monad OPC Trust Layer

## 问题陈述

AI 时代出现了大量“一人公司”或 AI 增强型个人创作者，但他们很难把能力转换成可被信任、可比较、可持续交易的商品。买方也很难判断一个知识资产是否真实、是否持续更新、是否来自可信 OPC、是否存在可追踪的订阅和声誉记录。

传统简历和普通内容平台无法证明真实交付能力；普通自由职业平台偏向人工撮合和一次性服务；纯 Web3 bounty 平台又不适合知识资产的持续订阅与版本演进。需要一个面向个人 OPC 的可信交易层，让可验证的知识资产可以被发现、订阅、版本化、留痕和评价。

## 解决方案

打造一个基于 Monad 的 OPC 知识订阅市场，让 Verified OPC 将可验证的知识资产以订阅方式出售给买家。平台围绕资产页而不是个人主页组织交易，支持公开注册但发布前审核，支持链下内容存储与链上可信记录，支持首期托管、首期退款、结构化反馈和平台人工裁决。

v1 只做内容访问，不做执行型服务。买方订阅后获得对当前有效版本的持续访问权；订阅到期后保留资产元数据和购买记录，但不保留正文访问。平台在 Monad 上记录资产存在、版本发布、订阅、评价和争议相关的可信事件。

## 用户故事

1. 作为一个个人 OPC，我想注册钱包并提交作品样本，通过审核后获得发布资格，这样我可以把知识资产卖给买家。
2. 作为一个个人 OPC，我想创建 OPC Profile，这样买家可以看到我的验证状态、代表作品和声誉摘要。
3. 作为一个个人 OPC，我想上传文档、模板方法论或数据包形式的知识资产，这样我可以把能力商品化。
4. 作为一个个人 OPC，我想为知识资产填写标题、摘要、分类、适用场景、更新频率和价格，这样买家可以比较不同资产。
5. 作为一个个人 OPC，我想给资产上传 Markdown 或 PDF 文件，并把正文存到链下存储，这样我可以控制公开内容范围。
6. 作为一个个人 OPC，我想在发布新版本时保留旧版本记录，这样订阅者可以持续获得当前有效版本。
7. 作为一个个人 OPC，我想让资产页成为主展示页，这样买家能直接从商品理解价值，而不是先看个人介绍。
8. 作为一个个人 OPC，我想让买家只能通过平台订阅，而不是直接联系我成交，这样市场结构更清晰。
9. 作为一个买家，我想先浏览资产页再决定是否订阅，这样我可以先判断内容是否符合需求。
10. 作为一个买家，我想看到资产的免费预览、样例页、摘要和作者验证状态，这样我可以降低订阅决策成本。
11. 作为一个买家，我想用 Monad 原生代币支付订阅费，这样支付结算能在链上完成。
12. 作为一个买家，我想订阅后立即获得内容访问权限，这样我可以直接使用当前有效版本。
13. 作为一个买家，我想在订阅期间持续访问同一资产的最新有效版本，这样我不会买到过时内容。
14. 作为一个买家，我想在订阅到期后保留购买记录和资产元数据，但失去正文访问，这样我能回看交易历史。
15. 作为一个买家，我想对资产提交结构化反馈，这样我的评价可以进入平台声誉体系。
16. 作为一个买家，我想在首期订阅发现内容明显不符时申请退款，这样我可以降低试错风险。
17. 作为一个买家，我想让平台人工裁决争议，这样我在内容不符时有申诉路径。
18. 作为一个买家，我想看到资产的订阅记录和声誉摘要，这样我可以判断其他人是否愿意继续购买。
19. 作为一个买家，我想按分类和搜索找到文档、方法论或数据包资产，这样我能快速定位需求。
20. 作为一个买家，我想先看到资产而不是作者个人主页，这样我可以直接比较商品价值。
21. 作为一个平台运营者，我想审核 OPC 的钱包注册、作品样本和人工材料，这样我能控制供给质量。
22. 作为一个平台运营者，我想只允许 Verified OPC 发布资产，这样垃圾内容不会快速淹没市场。
23. 作为一个平台运营者，我想对首期订阅款进行托管，这样退款和争议处理有抓手。
24. 作为一个平台运营者，我想按订阅收取平台抽成，这样市场有可持续商业模式。
25. 作为一个平台运营者，我想把资产、版本、订阅、评价和争议相关事件记录到 Monad，这样平台拥有可信交易叙事。
26. 作为一个评委，我想在一次演示里看到“发布一个知识资产、订阅一次、上链留痕、提交反馈”的闭环，这样我可以确认产品价值。
27. 作为一个评委，我想看到 Monad 不是支付壳，而是可信交易与声誉记录层，这样我可以判断链的必要性。
28. 作为一个评委，我想看到个人 OPC 可以把可验证知识资产做成订阅交易，这样我能理解这是 AI 时代的新型市场。

## 实现决策

- v1 的交易对象是 **Knowledge Asset**，而不是泛化的个人能力展示。
- Knowledge Asset 仅覆盖三种默认形态：**Document / Report**、**Template / Methodology**、**Dataset / Annotation Pack**。
- 资产出售的是 **Usage Right** 和 **Content Access**，不是所有权或转授权。
- 资产生产方式允许 **Human-authored**、**AI-assisted**、**Agent-executed**，但必须在资产元数据中披露。
- v1 只做 **Content Access**，不做执行型服务或代办结果交付。
- 交易模型采用 **Subscription Access**，默认按订阅持续访问，而不是永久买断。
- 订阅结算使用 **Monad Native Token**。
- 平台对每笔订阅收取 **Platform Fee**。
- 买方看到的主入口是 **Asset Page**，不是 OPC Profile。
- OPC Profile 作为辅助页面展示验证状态、代表作品和声誉摘要。
- 资产页的主 CTA 是 **Subscribe**。
- 买方可以订阅当前有效版本，活跃订阅者自动获得当前有效版本访问权。
- 订阅到期后保留 **Expired Access** 所需的元数据和购买历史，但不保留正文访问。
- 平台在链上记录 **Trust Event**，重点是资产存在、版本存在、订阅存在和声誉存在。
- 资产内容本体放在链下存储，只把 hash、URI 和版本记录留在链上。
- 平台只证明来源、版本和交易，不证明内容真伪或知识正确性。
- 只有通过审核的 **Verified OPC** 才能发布资产。
- Verified OPC 的入场条件是 **Verification Pack**，包含钱包注册、作品样本和人工审核。
- 作品样本以 **Portfolio Sample** 形式提交，用于证明持续产出能力，而不是证明身份。
- 平台采用 **Public Registration**：任何人都可以注册，但发布前必须审核。
- 发现方式优先使用分类和搜索，不做复杂 AI 推荐作为 v1 核心能力。
- 分类按交付形态组织，不按行业组织。
- 买家优先面向创业者和小团队，卖家优先面向独立研究者或顾问。
- 平台支持免费预览，但只开放摘要、样例页和验证状态，不开放完整正文。
- 平台支持首期订阅的 **Escrow**，用于退款和争议处理。
- 首期存在 **First-term Refund** 机制，仅在内容与描述明显不符时生效。
- 争议采用 **Platform Review**，由平台人工裁决。
- 声誉系统以 **Reputation Event** 和 **Structured Feedback** 为主，不做长评论社区作为 v1 核心。
- 第一版目标不是规模化 GMV，而是完成首个可验证的知识资产订阅成交。

## 测试决策

- 重点测试外部行为，不测试内部实现细节。
- 测试 Verified OPC 的准入流程：钱包注册、作品样本提交、人工审核通过后可发布。
- 测试资产创建与发布：能否创建 Document / Report、Template / Methodology、Dataset / Annotation Pack 三类资产。
- 测试资产页展示：买家是否优先看到资产信息、免费预览、验证状态和订阅按钮。
- 测试订阅流程：买家是否能用 Monad 原生代币完成订阅，是否生成可验证记录。
- 测试版本切换：活跃订阅者是否自动获得当前有效版本。
- 测试到期行为：订阅到期后是否仅保留元数据和购买历史。
- 测试托管与退款：首期托管是否能在争议时触发退款或保留款项。
- 测试反馈与声誉：结构化反馈是否会形成可追踪的 Reputation Event。
- 测试平台抽成：订阅成交后是否按设定比例抽取 Platform Fee。
- 参考同类行为测试模式：围绕可见页面、表单提交、状态切换和支付完成后的结果做集成测试，而不是验证内部状态字段。

## 不在范围内

- 不做个人技能撮合的自由职业平台。
- 不做招聘平台或简历平台。
- 不做执行型外包服务、代办服务或 SLA 交付市场。
- 不做完整 AI 推荐系统。
- 不做复杂的社区评论产品。
- 不做内容真伪验证或知识正确性鉴定。
- 不做全文链上存储。
- 不做永久买断、转售权或二次分发权。
- 不做 KYC 级别实名体系作为 v1 默认门槛。
- 不做社区仲裁、链上仲裁网络或司法级争议机制。

## 进一步说明

这个 PRD 面向黑客松和首版产品验证，核心不是做“大而全的知识市场”，而是证明一个最短闭环：Verified OPC 发布一个知识资产，买家完成一次订阅，平台在 Monad 上留下可验证的资产、版本、订阅与声誉记录。

如果 v1 成功，后续再扩展到更复杂的 Agent 服务、更多资产类型和更强的匹配机制。
